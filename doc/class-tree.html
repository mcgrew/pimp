Class Hierarchy

  • object: The most base type
    • wx._core.Object: The base class for most wx objects, although in wxPython not much functionality is needed nor exposed.
      • wx._core.EvtHandler: Proxy of C++ EvtHandler class
        • wx._core.Window: wx.Window is the base class for all windows and represents any visible object on the screen.
          • wx._windows.TopLevelWindow: Proxy of C++ TopLevelWindow class
            • wx._windows.Frame: Proxy of C++ Frame class
              • pimp.editor.Frame: The Frame object is used as the main application window.
              • wx._windows.SplashScreen: Proxy of C++ SplashScreen class
                • pimp.pimp.Splash: Create a splash screen widget.
        • wx._core.PyApp: The wx.PyApp class is an implementation detail, please use the wx.App class (or some other derived class) instead.
          • wx._core.App: The wx.App class represents the application and is used to:
            • pimp.pimp.App: wx.App subclass required by wxPython.
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • pimp.error.ImageFormatError: An exception to be raised in the event of a problem reading the file.
        • pimp.error.UnsupportedImageTypeError: An exception to be raised in the event that there is no plugin to handle.
        • pimp.error.ImageReadError: An exception to be raised in the event of a problem reading the file.
        • pimp.error.ExtensionError: An exception to be raised in the event that an extension malfuncions( returning invalid data, etc ).
  • pimp.image.Image: A class for storing image data.
  • wx._core.PyOnDemandOutputWindow: A class that can be used for redirecting Python's stdout and stderr streams.
